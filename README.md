# Angular Code Review Exercise

## Overview

A pull request has been opened by a junior developer adding a new feature to this Angular application. Your task is to review it as you would in a real team setting.

This exercise is not about finding a fixed list of problems — it is about how you read unfamiliar code, what you notice, how you prioritise your findings, and how you would communicate them to a colleague.

**Time limit: 15 minutes.**

## Setup

```bash
npm install
npm start
```

The application will be available at `http://localhost:4200`.

## The exercise

1. Open the open pull request in this repository.
2. Read through the changes introduced by the junior developer.
3. Prepare to walk the interviewer through your findings.

## What we are looking for

- **Thoroughness** — do you read beyond the surface?
- **Prioritisation** — do you distinguish between what blocks the merge and what is a nice-to-have?
- **Communication** — how would you deliver your feedback to the author in a way that is useful and constructive?
